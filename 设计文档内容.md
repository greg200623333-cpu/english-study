## 第一章 需求分析

### 1.1 项目背景

当前英语学习工具存在以下痛点：

**通用学习痛点**
- 传统学习平台缺乏趣味性，用户粘性低
- 题库资源分散，缺乏系统化管理
- 缺少智能化的学习进度追踪和个性化推荐
- 词汇记忆效率低，遗忘曲线未被有效利用

**计算机专业学生的特殊困境**
- 阅读英文技术文档、算法论文时词汇障碍严重，影响学习效率
- 技术面试、代码评审等场景需要专业英语表达，但缺乏针对性训练
- 现有英语学习平台与编程学习完全割裂，无法在语言学习中融入技术场景
- 口语表达能力弱，无法流畅参与英文技术讨论

### 1.2 目标用户

主要面向CET-4/6、考研英语备考学生，以及计算机、建筑学等专业学生的英语能力提升需求。

### 1.3 主要功能

1. **SSA战略勤务局** - 学习模式选择、法案系统、技能平衡、行政力与赤字管理
2. **作战部署台** - 科目练习入口，支持真题档案与AI模拟题
3. **词汇国库** - 词汇管理系统，分层记忆与复习机制
4. **作文战场** - AI辅助写作与评分反馈
5. **Portal专业工作台** - 算法阅读、技术文档分析、场景口语对话

### 1.4 竞品分析

| 产品 | 核心功能 | 优势 | 局限 | 我们的优势 |
|------|----------|------|------|------------|
| 扇贝/百词斩 | 词汇记忆 | 词库丰富、打卡机制 | 功能单一，缺乏完整学习闭环 | 完整学习闭环+专业模式 |
| 流利说 | 口语练习 | AI评分、场景对话 | 通用场景为主，无专业针对性 | 专业场景对话(技术面试、设计评审) |
| 考虫/有道 | 题库练习 | 真题资源丰富 | 缺乏游戏化设计，用户粘性低 | 游戏化+AI生成+专业模式 |
| Duolingo | 游戏化学习 | 趣味性强、留存率高 | 面向基础学习，不适合考试备考 | 游戏化+考试备考+专业场景 |

**差异化价值：** 国内首个将专业技术场景与英语学习深度融合的平台，通过游戏化设计提升用户粘性，通过AI技术降低内容成本，填补专业英语细分赛道的市场空白。

### 3.3 关键算法

#### 3.3.1 间隔重复算法(SRS)

基于SM-2算法改进，动态调整复习间隔：

**核心参数：**
- Stability(稳定性) - 表示记忆保持时间，范围0.1-365天
- Difficulty(难度) - 表示单词难度，范围1.0-3.0

**评级规则：**
```
响应时间 < 1.2秒 → Perfect
响应时间 < 3秒 → Good
响应时间 < 5秒 → Hard
响应时间 > 5秒 → Forgot
```

**更新公式：**
```
Perfect: stability = stability * 4, difficulty = max(1.0, difficulty - 0.15)
Good: stability = stability * 2, difficulty = max(1.0, difficulty - 0.05)
Hard: stability = min(stability * 1.2, 30), difficulty = min(3.0, difficulty + 0.2)
Forgot: stability = 0.1, difficulty = min(3.0, difficulty + 0.4)

next_review = now + stability * 24 * 60 * 60 * 1000
```

**毕业机制：** stability达到365天自动归档为"永久掌握"

#### 3.3.2 词汇GDP计算

```
GDP = Σ(word_difficulty × mastery_level)
mastery_level = stability / 365
```

量化学习成果，提升成就感，激励持续学习。

#### 3.3.3 AI题目生成

使用DeepSeek API的结构化输出(JSON mode)，确保题目格式规范：

**Prompt模板：**
```
生成一道{科目}的{题型}题目，难度为{难度}。
要求：
1. 符合{考试类型}的出题风格
2. 包含题目、选项、答案、解析
3. 以JSON格式输出
```

**质量保证：**
- AI生成后人工审核
- 优质题目进入题库
- 用户反馈机制

### 5.3 生产部署

**部署架构：**

本项目采用Next.js Standalone模式 + PM2进程管理 + 自动化部署脚本的方式部署到阿里云ECS服务器。

**部署目录结构：**
```
/www/wwwroot/english-study/
├── deploy/                    # 部署目录（运行时）
│   ├── server.js             # Next.js 服务器入口
│   ├── .next/static/         # 静态资源
│   ├── public/               # 公共资源
│   ├── .env.production       # 环境变量
│   └── logs/                 # 日志目录
├── backups/                  # 自动备份目录
├── deploy-server.sh          # 自动部署脚本
├── rollback.sh               # 回滚脚本
└── ecosystem.config.js       # PM2配置文件
```

**首次部署步骤：**

1. **服务器环境准备**
```bash
# 安装Node.js 18
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install 18
nvm use 18

# 安装pnpm和PM2
npm install -g pnpm pm2
```

2. **克隆项目到服务器**
```bash
cd /www/wwwroot
git clone <your-repo-url> english-study
cd english-study
```

3. **配置环境变量**
```bash
# 创建.env.production文件
nano .env.production
```

4. **执行自动部署脚本**
```bash
bash deploy-server.sh
```

**自动部署脚本功能：**

`deploy-server.sh`会自动执行以下步骤：
1. 环境检查（pnpm、node、pm2）
2. 备份当前部署（保留最近5个备份）
3. 清理旧构建
4. 安装依赖（pnpm install）
5. 构建生产版本（Next.js Standalone模式）
6. 准备部署文件（复制standalone、static、public）
7. 验证部署文件完整性
8. 停止旧服务并清理端口
9. 启动新服务（PM2 cluster模式）
10. 健康检查（HTTP响应测试）

**PM2配置说明：**

`ecosystem.config.js`配置项：
- 应用名称：ssa-app
- 运行模式：cluster（集群模式）
- 内存限制：1GB自动重启
- 日志路径：./logs/
- 环境变量：从.env.production加载
- Node参数：--max-old-space-size=2048

**更新部署：**
```bash
cd /www/wwwroot/english-study
git pull origin main
bash deploy-server.sh
```

**回滚到上一版本：**
```bash
bash rollback.sh
```

**常用PM2命令：**
```bash
pm2 status              # 查看服务状态
pm2 logs ssa-app        # 查看实时日志
pm2 restart ssa-app     # 重启服务
pm2 stop ssa-app        # 停止服务
```

### 6.4 遇到的困难与解决

**困难1：同设备切换账号时数据异常**
- 问题：新用户判断逻辑失效，导致不弹出签署动员令
- 解决：修复判断条件为"statusMap为空且store里的reviewDeficit === 0"
- 收获：深入理解了状态管理和数据持久化

**困难2：AI生成题目格式不稳定**
- 问题：AI有时返回非JSON格式
- 解决：使用DeepSeek的JSON mode，添加格式验证和重试机制
- 收获：学会了如何更好地设计Prompt和处理AI输出

**困难3：词汇复习算法调优**
- 问题：初版算法复习间隔不合理
- 解决：参考SM-2算法，根据用户反馈调整参数
- 收获：理解了认知科学在软件设计中的应用
