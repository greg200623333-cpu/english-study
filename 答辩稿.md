# 计算机设计大赛答辩稿 - 战略英语 WarRoom English

## 一、项目概述

### 1.1 项目名称
**战略英语 WarRoom English** - 基于AI的游戏化英语学习平台

### 1.2 项目背景
各位评委老师好，我们团队开发的是一款面向CET-4/6、考研英语备考以及计算机专业学生英语能力提升的智能学习平台。针对当前英语学习工具存在的痛点：

**通用学习痛点**
- 传统学习平台缺乏趣味性，用户粘性低
- 题库资源分散，缺乏系统化管理
- 缺少智能化的学习进度追踪和个性化推荐
- 词汇记忆效率低，遗忘曲线未被有效利用

**计算机专业学生的特殊困境**
- 阅读英文技术文档、算法论文时词汇障碍严重，影响学习效率
- 技术面试、代码评审等场景需要专业英语表达，但缺乏针对性训练
- 现有英语学习平台与编程学习完全割裂，无法在语言学习中融入技术场景
- 口语表达能力弱，无法流畅参与英文技术讨论

**其他专业学生的类似需求**
- 建筑学、设计类专业学生需要阅读英文设计规范、案例分析，同样面临专业词汇障碍

我们创新性地将"国家经营"游戏化机制与英语学习结合，并专门为计算机专业学生设计了 Portal 专业工作台，将算法阅读、技术文档分析、场景口语训练融为一体。同时，平台架构支持扩展到建筑学等其他专业领域，通过AI辅助和科学的记忆算法实现高效备考与专业能力提升。

### 1.3 核心创新点
1. **游戏化学习机制** - "国家经营"隐喻，将学习进度转化为GDP、行政力、赤字等可视化指标
2. **AI驱动的内容生成** - DeepSeek API动态生成模拟题、作文评分、智能推荐
3. **科学记忆算法** - 基于SM-2改进的间隔重复系统（SRS），自动化词汇复习调度
4. **沉浸式视觉体验** - 赛博朋克军事指挥部风格，提升用户参与感
5. **专业模式破解代码学习困境** - 国内首创将算法阅读、技术文档分析、编程场景口语训练融入英语学习，专为计算机专业学生设计

## 三、核心功能模块

### 3.1 SSA战略勤务局（核心学习系统）

**功能描述**
- 基于间隔重复算法的词汇记忆系统
- 自动化复习调度，根据遗忘曲线动态调整复习间隔

**技术实现**
我们实现了一套基于SM-2算法改进的间隔重复系统：

```typescript
// 核心计算逻辑
function calculateNextReview(current, rating) {
  switch (rating) {
    case 'perfect':  // 响应时间 < 1.2秒
      stability = stability * 4
      difficulty = max(1.0, difficulty - 0.15)
      break
    case 'good':     // 响应时间 < 3秒
      stability = stability * 2
      difficulty = max(1.0, difficulty - 0.05)
      break
    case 'hard':     // 响应时间 < 5秒
      stability = min(stability * 1.2, 30)
      difficulty = min(3.0, difficulty + 0.2)
      break
    case 'forgot':   // 响应时间 > 5秒
      stability = 0.1
      difficulty = min(3.0, difficulty + 0.4)
      break
  }
  next_review = now + stability * 24 * 60 * 60 * 1000
}
```

**创新点**
- **响应时间自动评级** - 根据用户翻卡速度自动判断掌握程度，减少操作负担
- **手动降级机制** - 用户可按K键主动标记难词
- **毕业机制** - stability达到365天自动归档为"永久掌握"

## 五、核心算法详解

### 4.1 间隔重复算法（SRS）

**算法原理**
基于Ebbinghaus遗忘曲线，动态调整复习间隔。我们的算法有以下特点：

1. **Stability（稳定性）** - 表示记忆保持时间，范围0.1-365天
2. **Difficulty（难度）** - 表示单词难度，范围1.0-3.0
3. **自动评级** - 根据响应时间自动判断掌握程度
4. **动态调整** - 根据评级调整下次复习时间

**优势**
- 自动化调度，无需用户手动规划复习时间
- 个性化难度调整，针对每个单词的掌握情况
- 长期记忆优化，stability最高可达365天

### 4.2 词汇GDP计算

**计算公式**
```
GDP = Σ(word_difficulty × mastery_level)
mastery_level = stability / 365
```

**意义**
- 量化学习成果，提升成就感
- 可视化进度，激励持续学习
- 多维度评估（Core词汇 vs Full词汇）

### 常见问题准备
1. **Q: 为什么选择游戏化设计？**
   A: 研究表明游戏化可以显著提升用户参与度和留存率。我们通过国家经营隐喻让学习变得有趣，将枯燥的刷题转化为战略游戏，用户更愿意持续学习。

2. **Q: Portal专业模式解决了什么问题？**
   A: 专业学生面临三重困境：专业文档看不懂、职场面试开不了口、英语学习与专业学习割裂。Portal通过算法阅读、场景对话、文档分析三个模块深度融合，让学生在学习专业知识的同时提升英语能力。这是国内英语学习平台的市场空白。

3. **Q: 建筑学功能具体是什么？**
   A: 建筑学模块包括：英文设计案例阅读（AI提取专业术语）、设计规范文档分析、设计评审场景模拟。基于AI的可扩展架构，我们可以快速适配新专业领域。

4. **Q: 相比竞品（扇贝、流利说、考虫）的优势是什么？**
   A: 
   - **扇贝/百词斩**：只做词汇记忆，我们有完整学习闭环+专业模式
   - **流利说**：通用口语，我们有专业场景对话（技术面试、设计评审）
   - **考虫/有道**：题库为主，我们有游戏化+AI生成+专业模式
   - **核心差异**：国内唯一将专业学习与英语学习深度融合的平台

5. **Q: AI如何保证题目质量？**
   A: 我们使用DeepSeek API的结构化输出（JSON mode），确保题目格式规范。同时建立人工审核机制，优质AI生成题目进入题库。

6. **Q: 间隔重复算法的效果如何？**
   A: 基于Ebbinghaus遗忘曲线，相比传统学习方法可提升记忆效率40-60%。我们的算法根据响应时间自动评级，无需用户手动操作，学习体验更流畅。

7. **Q: 如何保证用户数据安全？**
   A: 采用JWT认证、Bcrypt密码哈希、Supabase RLS行级安全策略，全站HTTPS加密。用户数据存储在Supabase云端，符合数据安全规范。
